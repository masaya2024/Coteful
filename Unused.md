# Conteful レビューと未実装候補

更新日: 2026-03-06

## 現状の整理

既存実装で成立していること:

- ログインとセッション維持
- プロジェクト切り替えと招待承認
- Content Model / Entry / Media の CRUD
- API Token / Webhook の基本管理
- 公開 API / 管理 API のモック構成

不足または弱い機能:

| 優先度 | 項目 | 判定 | 理由 | 対応 |
| --- | --- | --- | --- | --- |
| P1 | 永続化ストア未導入 | Defer | 現状は in-memory モックなので、再起動で変更が消える。MVP 方針上は許容だが、次段階では最優先の基盤課題。 | 保留 |
| P1 | 管理APIの認可境界強化 | Defer | セッション / project scope はあるが、管理系 CRUD にロール制御が薄い。今後の実運用前に要補強。 | 保留 |
| P1 | API 設定画面の Delivery API 実プレビュー | Approve | 現状は静的サンプル表示だけで、公開中 Entry が実際にどう返るか確認できない。開発者向け導線として価値が高い。 | 今回実装 |
| P1 | Delivery API 一覧エンドポイントの引数不整合修正 | Approve | 公開 API の一覧取得で `q` と `scope` の受け渡し順がずれており、検索と project scope が崩れる。機能不足というより実動作上の欠陥。 | 今回修正 |
| P2 | Media 一覧のタグフィルタ UI | Approve | 仕様上「検索・フィルタ」が欲しいが、現状は検索のみ。バックエンドは `tag` を受け取れるので低コストで補完できる。 | 今回実装 |
| P2 | Entry 一覧のステータスフィルタ | Approve with scope cuts | 下書き / 公開の絞り込みは有用。ただし今回は API プレビュー優先。 | 保留 |
| P2 | モデル定義に対する入力バリデーションの実行 | Approve with scope cuts | 型定義はあるが、UI では pattern 以外が薄く、Entry 保存時の実行も弱い。価値は高いが変更面が広い。 | 保留 |
| P3 | Webhook の編集 / pause-resume UI | Defer | API とストアには更新導線があるが、MVP としては新規作成・削除・テスト送信が先。 | 保留 |
| P3 | プロジェクト詳細ページとメンバー一覧 UI | Defer | データ構造はあるが、現状は切り替えと招待が優先。画面を増やすより既存導線強化が先。 | 保留 |

## 今回の実装方針

以下のみ着手する。

1. `/api-settings` に実データベースの Delivery API プレビューを追加する
2. Delivery API 一覧ルートの不整合を修正する
3. `/media` にタグフィルタを追加する

## 実装しないもの

今回は以下は入れない。

- バリデーション仕様の全面実装
- Webhook 編集画面の追加
- プロジェクト詳細画面の新設

理由:

- 画面と保存ロジックの両方へ波及し、今回の確認と実装スコープを超えるため
